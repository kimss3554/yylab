# LLM 위키

LLM이 단순 질의 응답 도구가 아닌 **지식 큐레이터**로서 마크다운 위키를 점진적으로 구축·유지하는 시스템 패턴.

## 핵심 원칙

**지식은 축적된다.** RAG가 매 질의마다 raw를 재탐색하는 반면, LLM 위키는 한 번 수집된 지식을 wiki/ 페이지에 보존하고 새 자료로 갱신한다.

## RAG와의 차이

| 항목 | RAG | LLM 위키 |
|------|-----|---------|
| 지식 축적 | ✗ (매번 재발견) | ✅ (위키에 누적) |
| 상호참조 | ✗ | ✅ (wikilink) |
| 모순 처리 | ✗ | ✅ (명시 기록) |
| 인간 역할 | 질문 | 선별·방향·질문 |

## 이 위키에서의 적용

이 저장소 자체가 LLM 위키 패턴의 실제 구현이다.

- `raw/관악구학교자료/` — 학교별 원본 자료 (불변)
- `wiki/` — LLM이 관리하는 입시정보 지식 베이스
- `AGENTS.md` + `.claude/` — 워크플로 스키마
