# LLM 위키 패턴

> 원본: `raw/llm-wiki-pattern.md` | 수집일: 2026-05-22 (원문 기재)

## 핵심 문제 의식

기존 RAG 방식(NotebookLM, ChatGPT 파일 업로드)은 **지식이 축적되지 않는다** — 매 질의마다 raw에서 재발견. 위키 패턴은 이를 해결한다.

## 핵심 아이디어

LLM이 쿼리 시 raw를 직접 검색하는 대신, **구조화·상호연결된 마크다운 위키**를 점진적으로 구축·유지한다.

- 새 소스 추가 시: 색인이 아닌 **읽기 → 핵심 추출 → 기존 위키 통합**
- 지식은 한 번 수집되어 최신 유지, 쿼리마다 재생성하지 않음
- 상호참조·모순 표시·종합이 이미 존재
- Obsidian = IDE, LLM = 프로그래머, 위키 = 코드베이스

## 아키텍처 3계층

| 계층 | 경로 | 역할 |
|------|------|------|
| 원본 자료 | `raw/` | 불변, LLM은 읽기만 |
| 위키 | `wiki/` | LLM 관리 마크다운 |
| 스키마 | `AGENTS.md` | 워크플로 정의 |

## 운영 방식

| 작업 | 절차 |
|------|------|
| **수집** | raw 추가 → LLM 처리 → sources/entities/concepts 갱신, log 기록 |
| **질의** | index → 관련 페이지 → 인용 답변 → 가치 있으면 outputs 저장 |
| **점검** | 모순·고립 페이지·오래된 주장·누락 링크 탐지 |

## 핵심 인프라

- **index.md**: 카탈로그, 중간 규모(~100소스, 수백 페이지)에서 RAG 없이도 충분
- **log.md**: 시간순 작업 로그

## 적용 사례

- 개인: 목표·건강·팟캐스트 메모
- 연구: 논문 → 발전하는 논지 위키
- **비즈니스**: 슬랙·회의록·고객 통화 → 내부 위키 ← *이 프로젝트 해당*

## 역사적 맥락

Vannevar Bush의 Memex(1945) — "문서 간 연결이 가치". 유지보수 문제는 LLM이 해결.
